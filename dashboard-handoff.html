Current Prototype Behavior

  • Left menu is split into top navigation and dashboard/favorites container. Original page remains untouched.
  • Dashboard list has an Ungrouped / Grouped switcher before All owners.
  • Ungrouped list has extra dashboards: Market Trends 2025, Customer Insights 2024, Revenue Forecasts Q3, Website Traffic Analysis 2024.
  • Collections column uses stack icon. One collection shows only stack icon; multiple collections show stack plus count such as +1; no collection shows no icon.
  • Grouped view is generated from the same dashboard collection membership model, not a separate static list.
  • Some dashboards intentionally belong to multiple collections and appear more than once in Grouped.
  • Dashboards without collection membership dissolve during Ungrouped to Grouped transition and do not appear in Grouped.
  • Grouped heart buttons toggle favorites and sync with Ungrouped rows plus the left dashboard list.
  • Libertex workspace dropdown opens but has no selection behavior yet.

Design Rules To Keep

  • Live Oboard product is the primary source of truth for spacing, icons, states, and component behavior.
  • Do not invent icons. Use existing Oboard SVG symbols already in the page.
  • Collections are labels/tags, not owners or permission containers.
  • A dashboard can belong to multiple collections.
  • Collection rows should not show collection owner or visibility as collection-level properties.
  • Empty heart outline appears only on row hover. Filled heart is visible in default and hover states.

Known Follow-ups

  • Transition from Ungrouped to Grouped may still need product-design review against live Oboard motion patterns.
  • Ungrouped right-side columns should be checked visually against the latest Figma/user screenshot after any future column changes.
  • Drag and drop in grouped empty collections needs careful target affordance if that flow continues.
  • Collection assignment popover should eventually match the production selector exactly, including search/create behavior.
