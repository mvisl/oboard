Latest Change Summary

  • Grouped collection badge clicks now highlight visible matching dashboard rows and collapsed collection header rows when matching dashboards are hidden inside collapsed collections.
  • Grouped collection header hover and dashboard-row hover both activate the shared flat-corner collection elevation; the header row itself must not add its own extra shadow. Only the outer top and bottom contour of the active collection extends full-width to the far left and far right edges; the internal divider under the collection header keeps the normal row offset. Adjacent collapsed collection dividers are suppressed so no short duplicate line remains above a hovered collection, and the grouped table header uses only one divider line.
  • Stable now carries the stronger grouped collection drag-preview shadow. Temp additionally has an experimental 70% opacity focus effect for non-hovered collections, controlled by GROUPED_DIM_OTHER_COLLECTIONS_ON_HOVER, a release animation that lets a dragged open collection glide into its final slot after mouseup, full-width normal dividers at the end of each visible collection plus under the grouped header, and 25% opacity internal grouped row dividers.
  • Grouped collection titles use regular weight; dashboard titles remain stronger.
  • Grouped badge hover/tooltip state is cleared after click and when the pointer leaves, preventing stuck hover-like rows.
  • The Ungrouped / Grouped switcher remains instant. The previous mode-switch animation is intentionally disabled.

Design Rules To Keep

  • Live Oboard product is the primary source of truth for spacing, icons, states, and component behavior.
  • Do not invent icons. Use existing Oboard SVG symbols already in the page.
  • Collections are labels/tags, not owners or permission containers.
  • A dashboard can belong to multiple collections.
  • Collection rows should not show collection owner or visibility as collection-level properties.
  • Empty heart outline appears only on row hover. Filled heart is visible in default and hover states.

Known Follow-ups

  • Collection bundle drag should be checked manually in a real browser after any grouped-row layout change.
  • Grouped empty collections still need careful target affordance if that flow continues.
  • Collection assignment popover should eventually match the production selector exactly, including search/create behavior.
